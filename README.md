# 飞书待办提醒

> 🤖 AstrBot 插件 — 自然语言创建/管理飞书待办，多级 DDL 自动提醒

---

## 功能

| 功能 | 说明 |
|------|------|
| 🗣️ 自然语言创建任务 | 发送文本或截图，LLM 自动提取标题、描述、截止时间、复杂度 |
| ❌ 自然语言删除任务 | 说出任务名称即可删除，自动匹配并清理飞书中的待办 |
| 🔔 多级 DDL 提醒 | 复杂任务提前 2 天+2 小时，普通任务提前 1 天+2 小时 |
| ✏️ 口头自定义提醒 | 支持"提前 N 天提醒我"等自定义提醒节点 |
| 🔄 定时巡检推送 | 每 30 分钟扫描一次，到期自动通过飞书 IM 提醒 |
| 👁️ 原生任务面板 | 所有任务在飞书客户端「任务」中统一管理，无需额外页面 |
| 🛡️ 防重复提醒 | 每条提醒规则仅触发一次，标记后不再重复推送 |

---

## 前置条件

- **AstrBot** ≥ 4.16
- 飞书开放平台创建自建应用，添加以下权限 scope：
  - `task:task:write` — 创建、编辑、删除任务
  - `task:task:read` — 查询任务列表
- 飞书应用已开启机器人能力
- 机器人与用户在同一租户

## 安装

### 方式一：插件市场安装

在 AstrBot WebUI 的「插件市场」中搜索「飞书待办提醒」，点击安装。

### 方式二：通过仓库链接安装

在 AstrBot WebUI 的「AstrBot 插件」页面中，使用「从仓库安装」，输入以下链接：

```
https://github.com/xBefore/astrbot_plugin_feishu_todo
```

安装后 AstrBot 自动加载插件，依赖 `httpx` 会在启动时自动安装。

## 配置

在 AstrBot WebUI 的「插件配置」中填写：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `feishu_app_id` | 飞书应用 App ID | `cli_xxxxxxxxxxxx` |
| `feishu_app_secret` | 飞书应用 App Secret | `xxxxxxxxxxxxxxxxxx` |

配置保存后无需重启，插件自动热加载。

## 使用

### 创建任务

在飞书中向机器人发送私聊消息，用自然语言描述任务：

```
下周五下午三点前完成项目报告，这个比较复杂，提前3天提醒我
明天下午六点去取快递
6月15日前完成学期论文，找人协作的
```

如果机器人接入的 LLM 支持多模态，直接发送截图等一切包含详细事件内容的信息即可，将自动提取任务信息。

### 删除任务

对机器人说删除指令即可：

```
删除之前那个"取快递"的任务
把"项目报告"那个待办删了
```

如果名称模糊匹配到多个任务，机器人会列出匹配项让你确认。

### 查看任务

所有任务可在飞书客户端**原生「任务」面板**中统一查看和管理。

## 提醒策略

| 任务类型 | 提醒节点 | 触发条件 |
|---------|---------|---------|
| 🔴 复杂任务 | 截止前 2 天 → 截止前 2 小时 | 涉及多人协作 / 多步骤 / 高出错风险 |
| 🟡 普通任务 | 截止前 1 天 → 截止前 2 小时 | 常规待办事项 |
| ✏️ 自定义 | 在上述基础上 + 截止前 N 天/小时 | 用户口头指定，如"提前3天提醒我" |

- 所有提醒通过飞书 IM 主动推送，每条只触发一次
- 截止前 2 小时的提醒使用飞书原生提醒机制，其余由插件巡检推送

## 架构

```
用户发送文本/截图
        ↓
AstrBot → LLM 多模态理解
        ↓
Function Call → 插件工具
        ├── create_reminder_task   ← 创建任务 + 写入 KV
        └── delete_reminder_task   ← 删除任务 + 清理 KV
        ↓
飞书 Task v2 API           定时巡检 (30min)
        ↓                        ↓
  创建/删除待办            匹配规则 → IM 推送提醒
```

## 常见问题

### 插件未加载

- 检查 `feishu_app_id` 和 `feishu_app_secret` 是否填写正确
- 查看 AstrBot 控制台日志，确认无报错

### Tool 未注册 / 对话无反应

- 确保 AstrBot ≥ 4.16
- 当前对话平台必须是飞书（lark），本插件仅支持飞书
- 确认 WebUI 中该工具已开启

### 创建任务失败

- 确认飞书应用已添加 `task:task:write` 和 `task:task:read` scope
- 确认 App ID / App Secret 正确，且应用已发布上线

### 提醒未推送

- 确认飞书应用具有消息发送权限
- 用户需要先主动向机器人发过消息（建立会话），插件才能回推

### 删除任务找不到

- 删除按标题关键词精确匹配，多个结果时会列出让用户选择
- 可以先用飞书原生「任务」面板确认任务标题
